# 订单图纸单入口上传和目录下载设计

## 背景

订单详情页目前把图纸上传拆成“上传文件”和“上传文件夹”两个入口，上传后的图纸用表格平铺展示。用户希望现场操作更简单：只看到一个上传入口，上传后按目录结构查看，并且点击文件或文件夹都能直接下载。

浏览器的原生文件选择器不能用同一个 input 同时选择文件和文件夹。本次采用一个可见上传入口加拖拽目录能力：用户点击“上传图纸”选择一个或多个文件；用户把文件夹拖到上传区域时，系统递归读取目录并保留路径。界面不再让用户在“文件/文件夹”之间做选择。

## 目标

- 订单详情页只保留一个可见的“上传图纸”入口。
- 点击上传入口可选择单个或多个文件。
- 拖拽文件或文件夹到上传区域可上传；文件夹递归读取并保留目录路径。
- 上传仍然使用覆盖模式：一次新上传替换该订单原有全部图纸。
- 图纸展示从表格改为目录树。
- 点击文件下载单个文件。
- 点击文件夹下载该目录下所有文件的 zip 包。
- 支持下载全部图纸 zip 包。
- 所有下载继续经过登录鉴权和工作区隔离。

## 非目标

- 不做浏览器原生“同一文件选择弹窗同时选文件和文件夹”，因为该能力不稳定且没有标准支持。
- 不做在线预览、文件夹重命名、移动文件、图纸版本历史或增量上传。
- 不引入压缩算法依赖；zip 使用无压缩存储格式，满足打包下载。

## 方案

### 上传交互

新增客户端组件 `OrderDrawingUpload`：

- 显示一个“上传图纸”按钮。
- 内部隐藏一个 `type=file multiple` input，用于点击选择文件。
- 上传区域监听 drag/drop。
- 拖拽文件夹时通过 Chromium/WebKit 的 `webkitGetAsEntry()` 递归读取目录。
- 提交时构造 `FormData`，每个文件使用相对路径作为 multipart filename，继续调用现有 `uploadOrderDrawingsAction`。

点击选择文件时不会选择文件夹；拖拽文件夹时支持目录。这个方案让页面只有一个入口，同时保留实际可用的文件夹上传能力。

### 目录树展示

订单详情页根据 `OrderDrawing.relativePath` 构建树：

- 根节点代表整套图纸。
- 目录节点按路径段聚合。
- 文件节点保留原始文件名、大小和下载链接。
- 目录节点链接到 zip 下载接口，prefix 为该目录路径。
- 根部提供“下载全部图纸”链接，prefix 为空。

### 打包下载

新增服务函数 `getOrderDrawingArchive(workspaceId, orderId, prefix)`：

- 按当前工作区和订单查询图纸。
- prefix 为空时打包全部。
- prefix 非空时只打包 `relativePath` 位于该目录下的文件。
- 文件内容从本地存储读取。
- 返回无压缩 zip Buffer 和下载文件名。

新增 API 路由 `GET /api/order-drawings/archive?orderId=...&prefix=...`：

- 未登录返回 401。
- 缺少 `orderId` 返回 400。
- 找不到订单、目录或文件返回 404。
- 成功返回 `application/zip`，并用 `Content-Disposition` 下载。

## 错误处理

- 上传空文件列表时沿用“请选择要上传的图纸文件”。
- 浏览器不支持目录拖拽时，仍支持点击选择多个文件。
- 文件夹 zip 内没有文件时返回 404。
- 下载接口不暴露本地文件真实路径。

## 测试

- 页面测试：订单详情只显示一个“上传图纸”按钮，不再显示“上传文件/上传文件夹”；目录树显示文件夹和文件链接。
- 路由测试：未登录不能下载 zip；登录后按工作区调用 archive 服务并返回 zip。
- 服务测试：按 prefix 打包指定文件夹；zip 中包含对应文件名，不包含其他目录文件。
- e2e 保持现有订单、机器和记录流程通过。
