# 订单价格、图纸和自动订单号设计

## 背景

当前 CNC 工厂管理应用已经支持登录、机器、订单、机器关联订单、从机器详情录入加工/出货记录，以及订单进度汇总。本次增量把订单信息补齐到更接近真实业务录入：订单号由系统自动生成，订单保存价格信息，订单详情支持上传图纸文件，并从机器页面快速跳转到关联订单。

这份规格覆盖当前版本的增量设计。早期 MVP 文档中把附件/图纸上传列为非目标；从本次需求开始，订单图纸进入当前实现范围。

## 目标

- 创建订单时不再要求用户填写订单号。
- 系统自动生成稳定、可读、唯一的订单号。
- 订单支持填写单价，并在列表和详情中显示订单金额。
- 订单详情支持上传单个文件、多个文件，或选择文件夹上传整套图纸。
- 用户重新上传图纸时，覆盖该订单原有的全部图纸。
- 图纸下载必须经过登录鉴权和工作区隔离。
- 机器详情页中当前关联订单和历史生产记录里的订单都可以点击跳转到订单详情。
- 设计保留后续多用户场景，避免并发创建订单或跨工作区访问图纸时出现数据串扰。

## 非目标

- 不做多员工账号、角色权限或审批流。
- 不做图纸在线预览、版本历史、差异比对或 CAD 文件解析。
- 不做云对象存储、断点续传或超大文件分片上传。
- 不做订单总价手工覆盖；订单金额只由单价乘计划数量计算。

## 方案选择

推荐方案是本地文件存储加数据库文件记录。

图纸文件保存到应用可配置的本地目录，默认位于 `storage/order-drawings/<workspaceId>/<orderId>/`。数据库保存每个图纸文件的原始文件名、相对路径、大小、MIME 类型和订单关系。Docker 部署时把该目录挂载为持久卷，避免容器重建丢失文件。

这个方案比直接把文件二进制存入 PostgreSQL 更适合当前 MVP：数据库保持轻量，下载逻辑清晰，也方便后续迁移到 S3、MinIO 或 NAS。缺点是需要管理文件目录和数据库记录的一致性；本次实现用“整单覆盖”的写入模型降低复杂度。

## 数据模型

### Orders

新增字段：

- `unitPriceCents`：可选整数，单位为分。空值表示未填写单价。

调整字段规则：

- `orderNo` 由服务端生成，不再从创建订单表单接收。
- 同一工作区内 `orderNo` 唯一。
- 自动订单号格式为 `ORD-YYYYMMDD-NNNN`，例如 `ORD-20260510-0001`。
- 递增序号按工作区和当天日期计算。
- 并发创建时依赖数据库唯一约束兜底，若生成号冲突则重新生成下一个序号。

派生显示字段：

- `unitPrice`：页面显示为人民币元。
- `orderAmount`：`unitPriceCents * plannedQuantity`，页面显示为人民币金额。

### OrderDrawings

新增订单图纸表。

关键字段：

- `id`
- `workspaceId`
- `orderId`
- `originalName`：用户上传时的文件名。
- `relativePath`：文件夹上传时保留的相对路径；普通文件上传时为文件名。
- `storedPath`：相对于图纸根目录的存储路径。
- `sizeBytes`
- `mimeType`
- `createdAt`

约束和规则：

- 所有记录携带 `workspaceId`。
- 图纸属于一个订单。
- 删除订单时级联删除图纸记录；实际文件清理由服务层在覆盖上传时处理。
- 同一个订单重新上传图纸时，先删除旧目录和旧数据库记录，再写入新文件和新记录。

## 文件上传行为

订单详情页提供两个上传入口：

- 上传文件：支持选择一个或多个文件。
- 上传文件夹：支持浏览器的目录选择能力，尽量保留目录结构。

两个入口提交到同一个服务端动作。服务端统一做这些处理：

- 必须登录，并只允许访问当前工作区订单。
- 至少上传一个非空文件。
- 清理文件名和相对路径，禁止 `..`、绝对路径和空路径段。
- 对重复相对路径自动加后缀，避免文件覆盖。
- 限制单文件大小、总文件数和总大小，防止误传超大目录拖垮本机。
- 以覆盖模式保存：本次上传成功后，订单只保留本次提交的图纸。

下载图纸通过受保护的 API 路由完成。路由根据当前登录用户的 `workspaceId` 查询图纸记录，确认归属后读取本地文件并返回下载响应。

## 页面和交互

### 订单列表

- 创建订单表单移除“订单号”输入。
- 创建订单表单新增“单价（元/件）”输入，允许留空。
- 列表继续显示订单号、客户、工件、交期、状态、计划、加工、出货和剩余。
- 列表增加单价和订单金额展示，未填写单价时显示 `-`。
- 搜索继续支持订单号和工件名称。

### 订单详情

- 基础信息中显示自动订单号、单价、订单金额、客户、工件、交期和备注。
- 新增“图纸文件”区域。
- 已上传图纸以列表展示文件名、相对路径、大小和上传时间。
- 每个图纸提供下载链接。
- 上传区域明确提示：重新上传会覆盖原有图纸。

### 机器详情

- “当前订单”文字改为链接，点击进入订单详情。
- 生产记录表里的订单文字改为链接，点击进入对应订单详情。
- 保持原有录入加工数量和出货数量的流程不变。

## 错误处理

- 单价格式错误时提示“单价最多保留两位小数”。
- 单价小于 0 时提示“单价不能小于 0”。
- 上传空文件列表时提示“请选择要上传的图纸文件”。
- 文件过大或数量过多时给出明确错误。
- 图纸不存在、跨工作区访问或未登录时，下载接口返回错误状态，不暴露真实文件路径。
- 自动订单号生成若遇到唯一冲突，服务端重试有限次数；仍失败时返回“订单号生成失败，请重试”。

## 测试

- 服务层测试自动订单号生成、同日递增、跨工作区隔离和唯一约束。
- 动作层测试创建订单时不接收用户订单号，单价解析为分。
- 服务层测试图纸覆盖上传会替换旧记录和旧文件。
- 路由测试未登录或跨工作区不能下载图纸。
- 页面测试订单列表和详情显示价格、图纸列表和覆盖提示。
- 页面测试机器详情里的订单链接指向 `/orders/[id]`。
- 端到端测试覆盖创建带价格订单、从机器页跳转订单详情的核心路径。
